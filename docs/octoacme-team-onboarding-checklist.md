# OctoAcme — Team Onboarding Checklist

## Purpose
Ensure new team members quickly understand their role, responsibilities, and how to collaborate effectively within the OctoAcme project structure.

## For All Team Members

### Week 1: Orientation
- [ ] Review [OctoAcme Project Management Overview](octoacme-project-management-overview.md)
- [ ] Read [OctoAcme Roles and Personas](octoacme-roles-and-personas.md) — understand all roles
- [ ] Review [Collaboration & Hand-off Guide](octoacme-collaboration-handoff-template.md)
- [ ] Get access to:
  - [ ] Project repository and documentation
  - [ ] Project board (GitHub Projects or equivalent)
  - [ ] Communication channels (Slack, Teams, etc.)
  - [ ] Meeting calendars
- [ ] Meet with Project Manager for project overview
- [ ] Attend team standup or sync meeting

### Week 2: Dive Deeper
- [ ] Review current sprint/iteration goals
- [ ] Understand Definition of Done (DoD)
- [ ] Review recent retrospective notes
- [ ] Shadow role-specific activities (see role sections below)
- [ ] Set up development environment (if applicable)

---

## Role-Specific Onboarding

### Developer
- [ ] Clone repository and set up local development environment
- [ ] Review coding standards and PR conventions
- [ ] Set up CI/CD pipeline access
- [ ] Complete first small task or bug fix
- [ ] Participate in code review
- [ ] Meet with Tech Lead for architecture overview
- [ ] Review test coverage and testing practices

### Product Manager
- [ ] Review product roadmap and backlog
- [ ] Understand success metrics and OKRs
- [ ] Meet with stakeholders to understand business context
- [ ] Review user research and personas
- [ ] Attend customer or user feedback session
- [ ] Review past release notes and feature launches

### Project Manager
- [ ] Review project charter and timeline
- [ ] Access and understand risk register
- [ ] Review current project status and metrics
- [ ] Meet with sponsor and key stakeholders
- [ ] Understand escalation paths
- [ ] Review communication cadence and reporting requirements

### Scrum Master
- [ ] Review team ceremonies schedule
- [ ] Understand team agreements and working norms
- [ ] Review velocity and team metrics
- [ ] Meet with team members individually
- [ ] Attend retrospective and planning sessions
- [ ] Understand team blockers and impediments

### QA Lead
- [ ] Review test strategy and quality standards
- [ ] Access test environments and tools
- [ ] Review defect tracking and triage process
- [ ] Understand quality gates and sign-off criteria
- [ ] Review test automation framework
- [ ] Meet with DevOps Engineer on CI/CD test integration

### UX Designer
- [ ] Review design system and component library
- [ ] Understand user personas and research findings
- [ ] Access design tools (Figma, Sketch, etc.)
- [ ] Review usability testing process
- [ ] Meet with Product Manager on roadmap priorities
- [ ] Review past design decisions and rationale

### Technical Writer
- [ ] Review documentation standards and style guide
- [ ] Access documentation platform and tools
- [ ] Understand documentation workflow and review process
- [ ] Identify documentation gaps and priorities
- [ ] Meet with Developers and Product Manager
- [ ] Review recent release notes

### DevOps Engineer
- [ ] Review infrastructure architecture
- [ ] Access CI/CD pipelines and deployment tools
- [ ] Understand monitoring and alerting setup
- [ ] Review incident response playbook
- [ ] Meet with security team on compliance requirements
- [ ] Understand deployment windows and release process

### Stakeholder
- [ ] Review project charter and business case
- [ ] Understand success metrics and ROI expectations
- [ ] Confirm communication cadence (monthly updates, etc.)
- [ ] Meet with Sponsor on strategic alignment
- [ ] Identify specific concerns or focus areas
- [ ] Clarify approval and decision-making authority

### Sponsor
- [ ] Review business case and expected outcomes
- [ ] Confirm budget and resource allocation
- [ ] Understand escalation criteria
- [ ] Meet with Project Manager and Product Manager
- [ ] Review stakeholder landscape
- [ ] Clarify decision-making authority and scope

## 30-60-90 Day Goals

### 30 Days
- Understand project context, goals, and current status
- Complete role-specific technical or process setup
- Begin contributing to daily work (within your role)
- Build relationships with key collaborators

### 60 Days
- Independently handle most day-to-day responsibilities
- Provide meaningful input in team discussions
- Identify opportunities for improvement
- Contribute to at least one significant deliverable

### 90 Days
- Fully autonomous in role responsibilities
- Mentor or onboard other team members
- Drive initiatives or improvements
- Demonstrate understanding of cross-functional dependencies
